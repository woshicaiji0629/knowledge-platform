配置IPv6回源

更新时间：
复制 MD 格式
产品详情
我的收藏

通过配置IPv6回源，CDN节点将支持以IPv6协议访问源站，并且支持配置使用不同的回源策略（例如：回源优先使用IPv6地址、回源跟随客户端协议版本、回源IPv4/IPv6负载均衡）。

背景信息

阿里云CDN大部分节点已支持使用IPv6协议访问源站。开启IPv6回源功能后，若源站支持IPv6协议，CDN节点将根据配置的回源策略使用IPv6协议回源。

说明

  • 目前中国香港、中国澳门、中国台湾及海外地域的CDN节点不支持IPv6回源。

  • 暂不支持纯IPv6源站的使用场景。如果源站全部使用IPv6地址，部分CDN节点可能无法回源。

操作步骤

  1. 登录CDN控制台。

  2. 在左侧导航栏，单击域名管理。

  3. 在域名管理页面，找到目标域名，单击操作列的管理。

  4. 在指定域名的左侧导航栏，单击回源配置。

  5. 在IPv6回源区域，单击编辑。

  6. 在IPv6回源对话框中，您可以选择开启优先使用IPv6、跟随客户端协议或IPv4/IPv6负载均衡。

    参数

    说明

    优先使用IPv6

    开启后，CDN回源侧将优先使用IPv6服务。

    说明

    CDN节点或源站不具备可用的IPv6地址时，将使用IPv4建连。

    跟随客户端协议

    开启后，CDN回源将跟随客户端请求的IP协议版本。

    说明

    • 客户端请求使用IPv6，则CDN优先回源IPv6源站，如果没有IPv6源站，则使用IPv4源站。

    • 客户端请求使用IPv4，则CDN优先回源IPv4源站，如果没有IPv4源站，则使用IPv6源站。

    IPv4/IPv6负载均衡

    开启后，不论客户端请求使用的是IPv4还是IPv6，都会使用轮询方式回源到各个源站地址。

    说明

    • IPv4/IPv6负载均衡功能与优先使用IPv6、跟随客户端协议这两个功能互斥，IPv4/IPv6负载均衡一旦开启，优先使用IPv6和跟随客户端协议这两个功能就会失效。

    • 如果IPv4、IPv6地址配置了权重比例，将按照权重比例回源。

  7. 单击确定。
