### 付费类型
付费类型影响实例的计费和收费规则，不同付费类型的实例遵循的资源状态变化规则也存在差异。

| 付费模式 | 说明 | 相关文档 |
| --- | --- | --- |
| 包年包月 | 先付费后使用 ，最短可以按周购买 。适用于长期稳定的业务，如 7*24 的 Web 服务、数据库服务等。 | [包年包月](../subscription.md) |
| 按量付费 | 先使用后付费，计费周期精确到秒，方便您按需购买和释放资源。适用于有大幅波动的场景，如临时扩展、测试、电商抢购等。 说明 推荐搭配使用 节省计划、 预留实例券优化成本。 | [按量付费](../pay-as-you-go-1.md) [什么是节省计划](../savings-plans.md) [什么是预留实例券](../reserved-instances.md) |
| 抢占式实例 | 先使用后付费，相对于按量付费实例能最高节约 90%的实例成本，但可能因市场价格变化或实例规格库存不足而自动释放实例。适用于无状态、容错能力强、中断容忍度高的业务场景，如测试、实时分析等。 | [抢占式实例](../spot-instance.md) |
